# 打鼾干预测试工具

基于 Electron + SerialPort 实现的跨平台多串口测试工具，用于智能床垫/睡眠设备的打鼾检测与干预功能验证。

## 功能特性

- ✅ **多串口支持** - 可同时连接 1-4 个串口设备，独立控制
- ✅ 串口自动检测和连接状态实时显示
- ✅ 自动加载内置鼾声数据文件
- ✅ 可配置采样率循环发送
- ✅ 实时计时显示（测试时长）
- ✅ 打印输出配置（串口选择、时间戳、十六进制格式）
- ✅ 简洁易用的图形界面
- ✅ 支持 Windows/Mac/Linux 全平台

**Windows 系统**：
1. Node.js 18+
2. Python 3.10+（微软商店安装）

**Mac/Linux 系统**：
1. Node.js 18+
2. Python 3.x

#### 打包命令

```bash
# 清理旧构建（推荐打包前执行）

# 重新编译原生模块
npm run rebuild

# ===== 按平台打包 =====

# Windows 64位 安装包（在 Windows 机器执行）

# Mac 安装包（在 Mac 机器执行）
npm run dist:mac

# Linux 安装包（在 Linux 机器执行）

# 打包目录（绿色版，不生成安装包）

npm run dist
```

#### 输出文件

## 采样频率配置

| 参数 | 值 | 说明 |
|------|-----|------|
| **采样频率** | **15 Hz** | 每秒发送 15 条数据 |
| **发送间隔** | **67 ms** | 1000ms ÷ 15 ≈ 67ms |
| **单轮数据** | 5001 条 | sn_5.txt 数据总量 |
| **单轮耗时** | 约 5分35秒 | 5001 × 67ms |
| **两轮耗时** | 约 11分钟 | 覆盖触发窗口 |

### 修改采样率

编辑 `src/main/index.js`，调整发送间隔：

```javascript
// 公式：间隔(ms) = 1000 ÷ 采样频率(Hz)
sendInterval = setTimeout(sendLoop, 67);  // 15Hz = 67ms
```

### 常用参考值

| 采样率 | 间隔 | 单轮耗时 | 适用场景 |
|--------|------|----------|----------|
| 1 Hz | 1000ms | 83分钟 | 慢速测试 |
| 5 Hz | 200ms | 16分钟 | 慢速验证 |
| **15 Hz** | **67ms** | **5分35秒** | **推荐默认** |
| 20 Hz | 50ms | 4分10秒 | 快速测试 |
| 30 Hz | 33ms | 2分45秒 | 超快速验证 |

## 使用说明

### 测试流程

1. **连接硬件**：将 1-4 个设备通过 USB 接入电脑
2. **选择串口**：点击「刷新串口」，为每个通道选择设备串口
3. **打开串口**：点击「打开串口」按钮（可一键打开所有已选串口）
4. **加载数据**：程序自动加载 `src/my_data/sn_5.txt`，也可手动选择
5. **配置打印**（可选）：
   - 选择打印输出串口（串口1/串口2/串口3/串口4）
   - 启用/禁用时间戳显示
   - 启用/禁用十六进制显示格式
6. **开始测试**：点击「开始写入」，所有已连接串口同步发送数据
7. **观察结果**：观察设备干预是否触发

### 多串口控制

- 每个串口通道独立选择、独立控制
- 「打开串口」一键连接所有已选择的串口
- 「开始写入」向所有已连接串口同步发送数据
- 「停止写入」暂停所有数据发送
- 「关闭串口」断开所有串口连接

### 数据文件格式

数据文件为纯文本格式，每行一条十六进制数据：

```
6D053A65F03CFA37
6D078D08F0EB4428
6D09C31BF171A359
...
```

## 常见问题

### Q: 串口打开失败？
- 检查设备是否正确连接
- 确认串口未被其他程序占用
- Windows 需安装对应串口驱动

### Q: 长时间不触发干预？
- 检查采样率设置是否过高
- 确认设备打鼾参数设置正确
- 尝试降低采样率，给算法足够检测时间

### Q: 如何更换测试数据？
- 将新的 txt 文件放入 `src/my_data` 目录
- 或点击「浏览」按钮直接选择文件

### Q: 多串口设备如何测试？
- 分别为每个通道选择对应的串口号
- 点击「打开串口」一次性连接所有设备
- 所有已连接串口会同步发送相同的测试数据
